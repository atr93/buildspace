# Contributing Guide

Thanks for your interest in contributing to **portfolio-lab**.

## Development Principles
- Keep all work public-safe (no secrets, credentials, or private endpoints).
- Prefer small, focused pull requests per project or feature.
- Write clear README docs for each project and include run instructions.
- Use mock/sample data for demos.

## Local Workflow
1. Fork and clone the repository.
2. Create a feature branch from `main`.
3. Make changes and run relevant checks.
4. Commit with clear messages.
5. Open a pull request with:
   - What changed
   - Why it changed
   - How it was tested

## Code Quality Expectations
- Clean naming and readable structure.
- Input validation and basic error handling.
- No overengineering for small projects.
- Keep dependencies minimal and justified.

## Safety Rules
- No offensive security tooling.
- No malware, exploit payloads, or target-specific recon.
- Security-adjacent projects must be educational/defensive and offline-safe.

## Documentation
Every project should include:
- Purpose
- Feature list
- Setup/run steps
- Sample data notes
- Future improvements
